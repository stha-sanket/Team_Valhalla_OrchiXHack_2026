\chapter{API Reference}
\label{app:api}
% Condensed from docs/06_api_reference.md

All REST endpoints live under the server base URL. Authentication uses
HTTP-only cookie JWTs (access + refresh); the WebSocket authenticates with
a short-lived ticket obtained over authenticated REST. ``User'' means any
signed-in account; ``admin'' requires the admin role.

\section{WebSocket Protocol}
The pathfinder channel is
\code{/ws/pathfinder?ticket=<JWT>\&visiting\_place\_id=<id>}. The client
sends \code{location} frames (\code{\{type, lat, long\}}) as geolocation
fixes arrive and \code{confirm\_visit} frames (\code{\{type, route\_id\}})
when the user confirms. The server replies to every location with a
\code{progress} frame --- next waypoint, distance, bearing, the place's
geofence radius, and \code{arrived}/\code{allVisited} flags --- and to every
confirmation with a \code{visit\_result} frame carrying \code{confirmed}
and, on rejection, a human-readable \code{reason}.
